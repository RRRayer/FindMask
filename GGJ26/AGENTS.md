# Repository Guidelines

## Project Structure & Module Organization
- `Assets/` holds all game content. Scenes live in `Assets/00. Scenes/` (e.g., `Game.unity`, `Lobby.unity`).
- Runtime code is in `Assets/01. Scripts/` grouped by feature (`UI/`, `Player/`, `Audios/`, `Settings/`, `SaveLoadSystem/`).
- ScriptableObjects are in `Assets/02. ScriptableObjects/` (events, audio, settings, pools).
- Prefabs are in `Assets/03. Prefabs/`, and audio clips are in `Assets/04. Audios/`.
- `ProjectSettings/` and `Packages/` define the Unity project and packages.

## Build, Test, and Development Commands
- Open the project in **Unity 6000.3.1f1** (see `ProjectSettings/ProjectVersion.txt`).
- Run locally via the Unity Editor Play button.
- Build via **File > Build Profiles** using `Assets/Settings/Build Profiles/Windows.asset` as the baseline profile.
- There are no scripted build or test commands in this repo yet; if you add one, document it here.

## Coding Style & Naming Conventions
- C# style matches the existing scripts: **Allman braces**, 4-space indentation, and PascalCase for types/methods.
- Fields use camelCase, often `[SerializeField] private` for inspector wiring.
- Keep new scripts in the appropriate feature folder under `Assets/01. Scripts/`.

## Testing Guidelines
- No automated tests are currently present (no `Assets/Tests/` directory).
- If you introduce tests, use Unity Test Runner and place them under `Assets/Tests/` (PlayMode) or `Assets/Editor/` (EditMode), and mention how to run them here.

## Commit & Pull Request Guidelines
- Git history currently contains only `Init Project`, so there is **no enforced commit convention**.
- Use short, imperative messages (e.g., “Add pause menu flow”).
- PRs should include: a brief summary, testing notes (what you clicked/ran), and screenshots or short clips for UI/gameplay changes.

## Configuration Tips
- Input actions are in `Assets/Settings/Input/` and should be regenerated if the `.inputactions` files change.
- Avoid editing `Library/` and `Temp/`—they are generated by Unity.
